# Codebase Structure

**Analysis Date:** 2026-01-19

## Directory Layout

```
document-research/
├── app/                    # Next.js App Router directory (pages, layouts, styles)
├── public/                 # Static assets (images, icons)
├── .claude/                # Claude Code configuration and GSD system
├── .planning/              # Planning and codebase documentation
├── .next/                  # Next.js build output (generated, not committed)
└── node_modules/           # Dependencies (generated, not committed)
```

## Directory Purposes

**app/:**
- Purpose: Application code using Next.js App Router
- Contains: React components, pages, layouts, global styles
- Key files: `app/layout.tsx`, `app/page.tsx`, `app/globals.css`

**public/:**
- Purpose: Static assets served at root URL
- Contains: SVG logos, images, static files
- Key files: `public/next.svg`, `public/vercel.svg`, `public/globe.svg`, `public/window.svg`, `public/file.svg`

**.claude/:**
- Purpose: Claude Code AI assistant configuration and GSD workflow system
- Contains: Agent definitions, commands, hooks, templates, settings
- Key files: `.claude/settings.json`, `.claude/agents/`, `.claude/commands/`, `.claude/get-shit-done/`

**.planning/:**
- Purpose: Project planning and codebase analysis documentation
- Contains: Codebase architecture docs generated by GSD system
- Key files: `.planning/codebase/ARCHITECTURE.md`, `.planning/codebase/STRUCTURE.md`

**.next/:**
- Purpose: Next.js build output and cache
- Contains: Compiled JavaScript, CSS, serverless functions, build artifacts
- Generated: Yes
- Committed: No

**node_modules/:**
- Purpose: NPM/pnpm dependencies
- Contains: Third-party packages and their dependencies
- Generated: Yes
- Committed: No

## Key File Locations

**Entry Points:**
- `app/layout.tsx`: Root layout component (wraps all pages)
- `app/page.tsx`: Home page component (root route)
- `package.json`: Project manifest and scripts

**Configuration:**
- `next.config.ts`: Next.js framework configuration
- `tsconfig.json`: TypeScript compiler configuration
- `eslint.config.mjs`: ESLint linting rules
- `postcss.config.mjs`: PostCSS/Tailwind CSS configuration
- `.gitignore`: Git ignore patterns

**Core Logic:**
- `app/page.tsx`: Home page UI component
- `app/layout.tsx`: Root layout with font loading

**Styling:**
- `app/globals.css`: Global styles and Tailwind imports

**Claude/GSD System:**
- `.claude/agents/gsd-*.md`: GSD agent definitions
- `.claude/commands/gsd/*.md`: GSD command definitions
- `.claude/get-shit-done/`: GSD workflow templates and references

## Naming Conventions

**Files:**
- TypeScript components: `.tsx` extension (e.g., `page.tsx`, `layout.tsx`)
- TypeScript non-components: `.ts` extension (e.g., `next.config.ts`)
- Stylesheets: `.css` extension (e.g., `globals.css`)
- Configuration: `.config.*` pattern (e.g., `eslint.config.mjs`, `postcss.config.mjs`)
- Static assets: Descriptive names with extensions (e.g., `next.svg`, `vercel.svg`)

**Directories:**
- Lowercase names (e.g., `app`, `public`, `.claude`)
- Dot-prefixed for tool/config directories (e.g., `.claude`, `.next`, `.planning`)
- Kebab-case for multi-word directory names (e.g., `get-shit-done`)

## Where to Add New Code

**New Feature (Page Route):**
- Primary code: `app/[route-name]/page.tsx`
- Tests: Not yet implemented (no test directory)
- Example: `app/about/page.tsx` for `/about` route

**New Feature (Nested Route):**
- Primary code: `app/[parent]/[child]/page.tsx`
- Layout: `app/[parent]/[child]/layout.tsx` (optional)
- Example: `app/dashboard/settings/page.tsx` for `/dashboard/settings`

**New Component/Module:**
- Shared components: Create `app/components/` directory
- Feature-specific: Co-locate with route (e.g., `app/dashboard/components/`)
- Implementation: Add `.tsx` file with default export

**Utilities:**
- Shared helpers: Create `app/lib/` or `app/utils/` directory
- Implementation: Add `.ts` files with named exports

**API Routes:**
- Route handlers: `app/api/[route-name]/route.ts`
- Example: `app/api/users/route.ts` for `/api/users`

**Styles:**
- Global styles: `app/globals.css` (existing)
- Component-specific: CSS Modules (`[name].module.css`) or Tailwind classes
- Tailwind config: Extend via `@theme` directive in CSS or configure Tailwind

## Special Directories

**app/:**
- Purpose: Next.js App Router application code
- Generated: Partially (`.next/` contains compiled output)
- Committed: Yes
- Convention: File-based routing where `app/` = `/`

**.next/:**
- Purpose: Next.js build output
- Generated: Yes (by `next build` or `next dev`)
- Committed: No (in `.gitignore`)
- Contains: Optimized assets, serverless bundles, build metadata

**.claude/:**
- Purpose: Claude Code AI assistant and GSD workflow system
- Generated: No (manual configuration)
- Committed: Yes
- Contains: Agent definitions, commands, hooks, templates

**.planning/:**
- Purpose: Project planning documentation
- Generated: Partially (by GSD codebase mapper)
- Committed: Yes (for documentation)
- Contains: Architecture, structure, and planning documents

**public/:**
- Purpose: Static assets
- Generated: No
- Committed: Yes
- Served at: Root URL (e.g., `public/next.svg` → `/next.svg`)
